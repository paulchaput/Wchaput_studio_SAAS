    The project detail page currently derives unique suppliers for OC PDF buttons by iterating line items and reading `li.suppliers?.id` (via the old proveedor_id join). After the schema change, supplier info lives in `li.line_item_costs[].suppliers`.

    Update the unique-supplier derivation logic:

    Old pattern (remove this):
    ```typescript
    const uniqueSuppliers = lineItems
      .filter(li => li.suppliers)
      .reduce((acc, li) => {
        const s = Array.isArray(li.suppliers) ? li.suppliers[0] : li.suppliers
        if (s && !acc.find(x => x.id === s.id)) acc.push(s)
        return acc
      }, [] as Array<{ id: string; nombre: string }>)
    ```

    New pattern (add this):
    ```typescript
    const uniqueSuppliers = lineItems
      .flatMap(li => (li.line_item_costs ?? []))
      .reduce((acc, cost) => {
        const s = cost.suppliers
        if (s && !acc.find(x => x.id === s.id)) acc.push({ id: s.id, nombre: s.nombre })
        return acc
      }, [] as Array<{ id: string; nombre: string }>)
    ```

    The OC PDF buttons and route handler references (`/proyectos/${project.id}/orden-compra?supplierId=${s.id}`) remain unchanged — only the supplier derivation logic changes.
