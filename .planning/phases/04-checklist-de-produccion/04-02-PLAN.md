    **Grouping logic** — group tasks by fase in canonical order using CHECKLIST_PHASES from lib/checklist-tasks.ts:
    ```typescript
    const FASE_ORDER: ChecklistFase[] = [
      'Comercial',
      'Diseño y Especificaciones',
      'Producción',
      'Entrega y Cierre',
    ]

    const grouped = FASE_ORDER.map(fase => ({
      fase,
      tasks: tasks.filter(t => t.fase === fase).sort((a, b) => a.sort_order - b.sort_order),
    }))
    ```

    **Optimistic state** — to avoid full-page flash on every status change (see Pitfall 4 in RESEARCH.md), maintain local task state:
    ```typescript
    const [localTasks, setLocalTasks] = useState<ChecklistTask[]>(tasks)
    ```
    On change: update local state immediately, then call the Server Action in the background. If action returns { error }, revert the change and optionally show an error.

    **Per-phase progress indicator** using calcPhaseProgress from lib/checklist-tasks.ts:
    ```typescript
    const progress = calcPhaseProgress(phaseTasks)
    // Render: "{progress.completed} / {progress.total} Completadas"
    ```

    **Per-task row layout:**
    Each task renders as a row with:
    1. Task name (text, left)
    2. Status Select — options: 'Pendiente', 'En Proceso', 'Completado', 'Bloqueado', 'N/A' — use native HTML select styled with Tailwind or Shadcn Select from @radix-ui/react-select (already installed)
    3. Assignee input — `<input type="text" placeholder="Responsable" />` with Tailwind classes from @/components/ui/input
    4. Due date input — `<input type="date" />` with same input styling

    **Status badge** — for visual differentiation, add a colored dot or badge beside the status:
    - 'Completado' or 'N/A': text-green-600 or text-muted-foreground
    - 'En Proceso': text-blue-600
    - 'Bloqueado': text-destructive (Shadcn destructive color)
    - 'Pendiente': text-muted-foreground

    **Action wiring** — for status changes, call updateChecklistTaskAction via FormData:
    ```typescript
    const handleStatusChange = async (task: ChecklistTask, newStatus: ChecklistStatus) => {
      // Optimistic update
      setLocalTasks(prev => prev.map(t => t.id === task.id ? { ...t, status: newStatus } : t))

      const fd = new FormData()
      fd.append('taskId', task.id)
      fd.append('projectId', projectId)
      fd.append('status', newStatus)
      const result = await updateChecklistTaskAction(fd)

      if (result?.error) {
        // Revert
        setLocalTasks(prev => prev.map(t => t.id === task.id ? { ...t, status: task.status } : t))
      }
    }
    ```

    For assignee and due_date, use onBlur (not onChange) to avoid firing on every keystroke:
    ```typescript
    const handleFieldBlur = async (task: ChecklistTask, field: 'assignee' | 'due_date', value: string) => {
      const fd = new FormData()
      fd.append('taskId', task.id)
      fd.append('projectId', projectId)
      fd.append(field, value)
      await updateChecklistTaskAction(fd)
    }
    ```

    **Empty state** — if `tasks.length === 0`, render a message: "Sin tareas de checklist. Crea un nuevo proyecto para generar el checklist automáticamente."

    **Layout** — use Card containers from @/components/ui/card for each phase group. Between phase groups, use @/components/ui/separator. Overall structure:
    ```
    <div className="space-y-6">
      {grouped.map(({ fase, tasks: phaseTasks }) => (
        <Card key={fase}>
          <CardHeader className="pb-3">
            <div className="flex items-center justify-between">
              <CardTitle className="text-base">{fase}</CardTitle>
              <span className="text-sm text-muted-foreground">
                {progress.completed} / {progress.total} Completadas
              </span>
            </div>
          </CardHeader>
          <CardContent>
            {/* task rows */}
          </CardContent>
        </Card>
      ))}
    </div>
    ```

    Import from @/components/ui/card, @/components/ui/separator, @/components/ui/input. Do NOT use react-hook-form for this component — the inline onChange/onBlur pattern with useState is simpler for a list of 30 independent fields.
